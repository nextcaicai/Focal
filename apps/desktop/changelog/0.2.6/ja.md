# v0.2.6 の新機能

## 新機能

- **ライブラリ検索**：サイドバーに検索を追加（今日 / 未読 / スターと並ぶ）。全購読を横断して記事を探せます。結果は関連度・日時・品質スコアで並び、翻訳タイトルにも一致します。
- **セマンティック検索**：Embedding を有効にすると、キーワード一致とベクトル類似度を組み合わせて検索します。言い換えや中英の表現の違いでもヒットしやすく、検索ヘッダーに索引の進捗を表示します。
- **既読・履歴もセマンティック索引へ**：Embedding は未読だけでなく既読・過去エントリにも付与されます（LLM の要約・タグ付けとは独立。対話モデルの追加コストはかかりません）。
- **AI タグ語彙の更新（形態 / 領域 / トピック）**：内容形態・領域・議題の三軸に対応。旧タグは対応表でオフライン変換し、全庫を LLM で打ち直す必要はありません。

## 改善

- キーワードの My Topics は、ベクトルがある場合タイトル部分一致だけでなくセマンティック類似でもマッチします。
- サイドバーの文言を「検索/Find」から「Browse（閲覧）」に揃え、ナビの意味を明確にしました。
- 購読が空のときは簡易追加モーダルではなく Discover へ誘導します。
- Embedding の進捗・再生成の説明を「埋め込み可能な全エントリ（既読を含む）」に合わせて分かりやすくしました。
- イベントリスナーと Hook 依存の整理で、UI の安定性と不要な再レンダーを改善しました。

## 修正

- Embedding ジョブキューの重複排除：同一記事が何度もキューに入り「待機」数が膨らむ・再計算が増える・操作が重くなる問題を修正しました。
- ライブラリ検索セッションから未使用の scope/sort を削除し、状態の取り残しを減らしました。
- 検索・リスト周りのローカライズを補完・統一しました（en / zh-CN / zh-TW / ja / fr-FR）。

## 謝辞

v0.2.6 をご利用いただき、フィードバックをお寄せいただきありがとうございます。ライブラリ検索とセマンティック索引は、実際のライブラリほど精度が上がります。
