# v0.2.6 の新機能

## 新機能

- **ライブラリ検索**：サイドバーに検索を追加（今日 / 未読 / スターと並ぶ）。全購読を横断して記事を探せます。結果は関連度・日時・品質スコアで並び、翻訳タイトルにも一致します。
- **セマンティック検索**：Embedding を有効にすると、キーワード一致とベクトル類似度を組み合わせて検索します。言い換えや中英の表現の違いでもヒットしやすく、検索ヘッダーに索引の進捗を表示します。全庫キーワード検索はベクトルなしでも常に利用できます。
- **AI タグ語彙の更新（形態 / 領域 / トピック）**：内容形態・領域・議題の三軸に対応。旧タグは対応表でオフライン変換し、全庫を LLM で打ち直す必要はありません。

## 改善

- キーワードの My Topics は、ベクトルがある場合タイトル部分一致だけでなくセマンティック類似でもマッチします。
- サイドバーの文言を「検索/Find」から「Browse（閲覧）」に揃え、ナビの意味を明確にしました。
- 購読が空のときは簡易追加モーダルではなく Discover へ誘導します。
- **セマンティック索引はデフォルト未読のみ**：Embedding は未読エントリ向け（BYOK と整合）。進捗と「未読索引の再構築」の文言を明確にし、既読履歴はキーワード検索のまま使えます。
- **Embedding のバッチ処理**：セマンティック索引をまとめて処理でき、構築が速く・負荷が抑えられます。
- **起動の軽量化**：データを段階的に読み込み、起動後いち早くメイン画面を使えるようにしました。
- **検索結果の見やすさ**：ヘッダーに実際のヒット数を表示。結果リストに上限を設け、大きなライブラリでもスムーズです。
- **短い語・固有名の検索**：人名や製品名などの短いクエリはキーワード一致を優先し、不要なベクトル検索を減らします。
- **リストの空状態コピー**：検索ヒットなしは「一致するコンテンツがありません」。「今日」「すべての未読」が空のときは「ここはしばらく静かです」を表示し、「未読ゼロ」の誤表示をやめました。
- イベントリスナーと Hook 依存の整理で、UI の安定性と不要な再レンダーを改善しました。

## 修正

- Embedding ジョブキューの重複排除：同一記事が何度もキューに入り「待機」数が膨らむ・再計算が増える・操作が重くなる問題を修正しました。
- ライブラリ検索セッションから未使用の scope/sort を削除し、状態の取り残しを減らしました。
- 検索・リスト周りのローカライズを補完・統一しました（en / zh-CN / zh-TW / ja / fr-FR）。
