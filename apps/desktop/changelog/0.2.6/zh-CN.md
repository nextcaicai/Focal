# v0.2.6 更新内容

## 新功能

- **全库搜索**：侧边栏新增搜索入口（与今天 / 未读 / 收藏并列），可跨全部订阅检索文章；结果按相关度、时间与质量分排序，支持译文标题匹配。
- **语义搜索**：在开启 Embedding 后，搜索同时使用关键词与向量相似度——同义说法、中英换词也能召回；搜索标题下展示语义索引进度。全库关键词搜索始终可用，不依赖向量。
- **AI 标签词表升级（形态 / 领域 / 议题）**：标签体系支持内容形态、领域与议题主题三轴；旧标签会按映射表离线整理，避免为全库重跑打标。

## 改进

- 关键词主题（My Topics）在具备向量时，会结合语义相似度匹配相关文章，而不只看标题字面。
- 侧边栏文案由「查找」统一为「浏览」，导航语义更一致。
- 无订阅时空状态改为直接进入 Discover 发现页，去掉临时的简易添加弹窗。
- **语义索引默认仅未读**：Embedding 只为未读条目建索引（与 BYOK 一致），进度与「重建未读索引」文案更清晰；已读历史仍可被关键词搜到。
- **Embedding 批量处理**：语义索引支持批量请求，建索引更快、更省资源。
- **启动更轻**：数据分阶段加载，打开应用时主界面更早可用。
- **搜索结果更清晰**：标题显示真实命中数量；结果列表有合理上限，超大库下也更流畅。
- **短词 / 实体名搜索更稳**：人名、产品名等短查询优先关键词匹配，减少无效的向量检索。
- **列表空状态文案**：搜索无结果显示「暂无匹配内容」；「今天」「全部未读」为空时显示「这里暂时很安静」，不再误显示「全部已读」。
- 多处事件监听与 Hook 依赖整理，界面交互更稳、重渲染更干净。

## 问题修复

- Embedding 任务队列去重：避免同一文章被反复入队导致「排队」虚高、重复计算与界面卡顿。
- 库搜索会话状态精简，去掉无用的范围/排序字段，减少异常状态残留。
- 搜索与列表相关本地化文案补齐与统一（中 / 英 / 日 / 繁中 / 法）。

## 致谢

感谢你使用 v0.2.6 并持续反馈——全库搜索与语义索引尤其需要真实库数据来校准体验。
