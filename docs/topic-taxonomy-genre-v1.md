# 内容形态词表 v1（Genre · 轴 A）

> 状态：方案真源；**已接入代码**（`ENTRY_CONTENT_TYPE_CANDIDATES` + `ENTRY_TAXONOMY_VERSION = 1`）  
> 范围：仅 **轴 A · 内容形态**（9 词，闭集、**单选**）  
> 问题：这篇 **怎么写 / 以什么体裁被消费**？（不是关于什么、不是哪个领域）  
> 配套：`topic-taxonomy-domain-v1.md`（B）、`topic-taxonomy-topics-v1.md`（C）

## 1. 使用规则

| 规则   | 说明                                                            |
| ------ | --------------------------------------------------------------- |
| 闭集   | 只能从下表 9 词选 1 个                                          |
| 单选   | 每文 **恰好一个** 主形态                                        |
| 兜底   | 低置信 / 体裁不清 → `其他`                                      |
| 正交   | **不**用形态表达议题或领域（「论文」是体裁，不是 C 里的议题词） |
| 与 B/C | A 不决定 B/C；同一形态可配任意领域与议题                        |
| 置信   | ≥0.75 写入；0.55–0.75 仅无强竞争时；&lt;0.55 → `其他`           |

### 展示（与 B/C 文档一致）

| 表面              | 轴 A                                                 |
| ----------------- | ---------------------------------------------------- |
| 列表行            | 空间够时 **可露 1 枚**（改变阅读预期：教程 vs 快讯） |
| 详情顶            | **建议默认露出**（与 C 并列，A 在前或后按视觉规范）  |
| 筛选 / 主题内过滤 | 完整提供（设计文档中的 contentType filter）          |
| 与 B              | 不同时把 B 和 A 都当「次要重复信息」；A 优先于 B     |

chip 争位：**C > A > D > B**。

### 相对现有代码

当前实现候选：`快讯 | 合集 | 教程 | 实测 | 分析 | 观点 | 论文 | 其他`。  
本表 **新增 `发布`**（从「短讯/通稿」里拆出正式发布体），接入代码时再改常量；在此之前可用规则：正式发布通稿暂标 `快讯` 或 `分析`（看篇幅）。

## 3. 正反例表

### 1. 快讯

**定义：** **短篇幅事实通报**——发生了什么，几乎不解释机制、不教步骤、不是完整发布说明文。

|                | 例子                                                                                 |
| -------------- | ------------------------------------------------------------------------------------ |
| **正例**       | 一句话/短段新闻；股价/排名/日程速递；「XX 今日开源」仅标题级信息                     |
| **反例**       | 完整 Release Notes / 发布博客 → `发布`；长文解读发布意义 → `分析`；多条打包 → `合集` |
| **冲突时优先** | vs 发布：有正式发布结构（特性列表、版本、获取方式）→ 发布；只有事件一句 → 快讯       |

---

### 2. 发布

**定义：** **正式宣告某物可用或更新**——产品/模型/功能/版本发布稿、更新日志体、官方 announcement（可长可短，但体裁是「发布」）。

|                | 例子                                                                                                                                                          |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **正例**       | 官方模型发布博文；Changelog；「Introducing XX」；应用商店级更新说明                                                                                           |
| **反例**       | 第三方「XX 发布了」短讯无说明 → `快讯`；发布后的深度技术拆解 → `分析`；发布后的横评 → `实测`                                                                  |
| **冲突时优先** | vs 分析：文中 50%+ 是背景与影响论述而非特性清单 → 分析；50%+ 是「出了什么、怎么用/获取」→ 发布。vs 教程：发布文附带 quickstart 仍算发布；整篇是 how-to → 教程 |

**目录价值：** 主题卡「模型发布 / 产品更新」主要靠 A=`发布`（+ B/C/实体过滤），不必再占 C 词表。

---

### 3. 合集

**定义：** **多条目聚合**——日报、周报、链接摘要汇编、newsletter digest；主线不是单一论证。

|                | 例子                                                                         |
| -------------- | ---------------------------------------------------------------------------- |
| **正例**       | 「今日 AI 30 条」；周刊精选；多链接短摘要串联                                |
| **反例**       | 单一长文分很多小标题但仍是一个论证 → `分析`；一个发布里列 10 个特性 → `发布` |
| **冲突时优先** | 有统一叙事主线的「系列盘点」→ 分析或观点；无主线堆条目 → 合集                |

**与 B/C：** 合集常 B=`其他` 或默认域，**C 宜空或 ≤2 个整篇主线**（见 C 文档）。

---

### 4. 教程

**定义：** **教会读者完成某事**——步骤、操作、配置、可复现路径；成功标准是「能跟着做」。

|                | 例子                                                                                            |
| -------------- | ----------------------------------------------------------------------------------------------- |
| **正例**       | 从零搭建 RAG；提示词/工作流 step-by-step；环境安装指南                                          |
| **反例**       | 只讲原理无步骤 → `分析`；跑分对比无教学 → `实测`；官方「What's new」→ `发布`                    |
| **冲突时优先** | vs 实测：目标是教学路径 → 教程；目标是结论「谁更好/快多少」→ 实测。可含少量测数据的教程仍归教程 |

---

### 5. 实测

**定义：** **用动手结果说话**——评测、对比、体验记录、benchmark 实操复现；主线是证据与结论，不是教学步骤。

|                | 例子                                                                                                                   |
| -------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **正例**       | 模型横评；工具一周体验；复现论文指标；「我测了 10 个 TTS」                                                             |
| **反例**       | 只转发官方分数无自测 → 更可能 `快讯`/`分析`；教别人怎么测 → 偏 `教程`；纯方法学谈榜是否公平 → `分析`（C 可评测与基准） |
| **冲突时优先** | vs 分析：数据服务于机制解释且自测很少 → 分析；自测/对比占核心 → 实测                                                   |

---

### 6. 分析

**定义：** **解释与结构化论述**——机制、因果、背景、比较框架、深度解读；不要求跟做，也不以「我表态」为主。

|                | 例子                                                                                              |
| -------------- | ------------------------------------------------------------------------------------------------- |
| **正例**       | 架构深拆；产业逻辑长文；政策影响分析；「如何理解这次更新」                                        |
| **反例**       | 强烈立场社论 → `观点`；步骤向 → `教程`；以表格跑分主 → `实测`                                     |
| **冲突时优先** | vs 观点：第三人称机制/证据主 → 分析；第一人称立场/倡导主 → 观点。边界文看标题与结尾是否在「站队」 |

---

### 7. 观点

**定义：** **表达看法、立场、评价或人物声音**——评论、倡导、随笔、访谈中的主张；读者消费的是「怎么看」而非「怎么做/出了什么」。

|                | 例子                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------- |
| **正例**       | 专栏评论；「我认为 Agent 被高估」；播客/访谈文字（表达向）；署名立场文                    |
| **反例**       | 中立机制科普 → `分析`；人物专访但整篇是产品发布会记录 → `发布` 或 `分析`                  |
| **冲突时优先** | vs C `人物与访谈`：**可双层**——A=观点（或分析），C=人物与访谈。A 不承担「是不是人物专题」 |

---

### 8. 论文

**定义：** **学术研究体裁**——论文、预印本、以论文结构（方法/实验/结论）为主的研究写作；或极贴近论文的学术综述。

|                | 例子                                                                                               |
| -------------- | -------------------------------------------------------------------------------------------------- |
| **正例**       | arXiv 论文；会议 paper；学术式 related work + 实验报告                                             |
| **反例**       | 「解读某论文」的博客 → `分析`；把论文当新闻说一句 → `快讯`；工业技术博客非学术结构 → `分析`/`发布` |
| **冲突时优先** | 博客读论文 → 分析；原文/强学术体 → 论文                                                            |

**注意：** C 词表 **不再**使用「论文」作议题；学术向用 A=`论文` + B/C（如 AI 与模型、大模型基础）。

---

### 9. 其他

**定义：** 体裁无法稳妥落入 1–8，或模型低置信。

|                | 例子                                        |
| -------------- | ------------------------------------------- |
| **正例**       | 体裁混杂且无主；诗歌/漫画等词表外；空内容壳 |
| **反例**       | 懒得标的分析文 → 仍应 `分析`                |
| **冲突时优先** | 最后选择                                    |

## 4. 近邻易混速查

| 易混对          | 一句话分开                     |
| --------------- | ------------------------------ |
| 快讯 ↔ 发布     | 事件一句 vs 正式发布说明       |
| 发布 ↔ 分析     | 出了什么 vs 为何重要/如何工作  |
| 发布 ↔ 教程     | 宣告与说明 vs 跟做路径         |
| 合集 ↔ 分析     | 多条堆叠 vs 单一论证           |
| 教程 ↔ 实测     | 教会做 vs 测出谁好             |
| 实测 ↔ 分析     | 证据结论 vs 机制解释           |
| 分析 ↔ 观点     | 解释框架 vs 立场表达           |
| 论文 ↔ 分析     | 学术体/原文 vs 解读文          |
| 观点 ↔ 人物向 C | 形态 vs 是否人物专题（可叠加） |

## 6. 明确不单列

| 不单列          | 归入                                       |
| --------------- | ------------------------------------------ |
| 评测 / 横评     | 实测（方法学长文无自测 → 分析）            |
| 周报 / 日报     | 合集                                       |
| 官方博客        | 发布 或 分析（看结构）                     |
| 访谈 / 播客     | 观点（+ C 人物与访谈）                     |
| 清单 / Listicle | 偏信息堆叠 → 合集或分析；偏教程步骤 → 教程 |
| 视频脚本转写    | 按文本主形态判，不单开「视频」             |
