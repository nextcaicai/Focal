# 领域词表 v1（Domain · 轴 B）

> 状态：方案真源；**已接入代码**（`ENTRY_DOMAIN_CANDIDATES`，库字段 `domain` / `domain_confidence`）  
> 范围：仅 **轴 B · 领域**（7 词，闭集、**单选**）  
> 配套：议题主题见 `topic-taxonomy-topics-v1.md`（轴 C）  
> 原则：B 回答「这篇落在哪个大盘」；C 回答「在 AI 向议题树里的哪几片叶子」；二者正交，C 可空。

---

## 1. 使用规则

| 规则        | 说明                                                                        |
| ----------- | --------------------------------------------------------------------------- |
| 闭集        | 只能从下表 7 词中选 1 个；集外丢弃                                          |
| 单选        | 每文 **恰好一个** 主领域；跨领域只看篇幅/标题主线                           |
| 兜底        | 不像任何桶 → `其他`；合集无主线 → 优先 `其他`（或库级默认域，产品策略另定） |
| 与 C 的关系 | B 不替代 C；非 AI 文常见 **B 有值、C 为空**                                 |
| 与 A 的关系 | 形态（教程/论文/快讯）不决定领域                                            |
| 与 D 的关系 | 实体（OpenAI 等）不决定领域（OpenAI 设计招聘文仍可能是设计/商业）           |
| 置信        | ≥0.75 写入；0.55–0.75 仅无强竞争时写入；&lt;0.55 → `其他`                   |

### 和轴 C 同名/近义时

| 轴 B       | 轴 C 近义           | 分工                                                                                                                         |
| ---------- | ------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| 商业与产业 | C3 `商业与产业`     | B=整篇大盘是商业；C=可关注的「商业向议题」切片。AI 公司融资：B 常=`商业与产业` 或 `AI 与模型`（看主线），C 可打 `商业与产业` |
| 社会与政策 | C3 `监管与政策`     | B=公共规则/社会议题大盘；C=`监管与政策` 偏 AI/科技监管专题，可双层同时有                                                     |
| 产品与工程 | C1 编码/部署/Agent… | B=产品与工程实践大盘；C=具体技术叶子                                                                                         |

**不要**因为 C 已有「商业与产业」就从 B 删掉——B 还要覆盖 **非 AI 商业** 与 **无 C 标签** 的文。

### 展示（复述，与 C 文档一致）

- 列表/详情：**默认弱展示或不展示 B**（库内 AI 占比高时噪音大）
- 筛选器 / 统计 / 「非 AI 浏览」：**完整提供 B**
- chip 争位：C > A > D > **B**

## 2. 词表总览（7）

| #   | 标签       | 一句话                                           | 典型供给                                 |
| --- | ---------- | ------------------------------------------------ | ---------------------------------------- |
| 1   | AI 与模型  | 模型、算法、能力与训练/推理技术本身              | 模型发布、论文、架构、评测技术向         |
| 2   | 产品与工程 | 做出来、用起来、运维起来：产品、工具、工程与交付 | 开发工具、上线、效率产品、工程实践       |
| 3   | 商业与产业 | 钱、公司、市场、竞争与商业化                     | 融资、财报、战略、行业格局               |
| 4   | 设计与体验 | 交互、视觉、设计系统、体验与品牌表达             | UX/UI、设计规范、体验研究                |

## 3. 正反例表

### 1. AI 与模型

**定义：** 主线是 **模型与智能技术本身**——能力、结构、训练、推理机制、基准上的技术进展，而不是「哪家公司融了多少钱」或「怎么做一个按钮」。

|                | 例子                                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **正例**       | 新基座/推理模型技术解读；架构与训练方法；对齐/安全的技术论文；多模态/Agent **机制** 深挖                                                                     |
| **反例**       | Cursor 改版与定价 → `产品与工程` 或 `商业与产业`；只谈融资与估值 → `商业与产业`；纯 UI 改版无模型 → `设计与体验`；「AI 改变写作习惯」文化评论 → `人文与生活` |
| **冲突时优先** | vs 产品与工程：主线是能力/原理 → AI；主线是产品功能与工作流 → 产品与工程。vs 商业：发布会里 80% 讲模型 → AI；80% 讲生态与收入 → 商业                         |

**轴 C：** 通常会有 C1（大模型基础、推理、多模态…）；允许双标多 C。

---

### 2. 产品与工程

**定义：** 主线是 **建造与交付**——产品功能、开发工具、工程实践、部署运维、落地实施过程（可以涉及 AI，但不以模型科学为主）。

|                | 例子                                                                                                                                                     |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **正例**       | IDE/协作工具更新；RAG 上生产的工程复盘；推理服务降本实践；内部平台与 SDLC；「我们如何做 AI 功能」的产品工程文                                            |
| **反例**       | 纯模型论文 → `AI 与模型`；纯视觉规范与体验原则无交付 → `设计与体验`；行业解决方案软文偏客户价值故事 → 可 `产品与工程` 或偏场景时仍选本类（比硬塞 AI 好） |
| **冲突时优先** | vs AI：见上。vs 设计：改交互规范/体验原则 → 设计；改功能与架构交付 → 产品与工程。vs 商业：功能发布附带定价一句 → 产品；整篇商业模式 → 商业               |

**轴 C：** 常有 `编码与开发` / `部署与推理工程` / `应用与落地` / `工具与 MCP` 等；也可 C 空（非 AI 工程文）。

---

### 3. 商业与产业

**定义：** 主线是 **经济行为与产业格局**——融资、并购、收入、定价、竞争、组织战略，不是技术说明书。

|                | 例子                                                                                       |
| -------------- | ------------------------------------------------------------------------------------------ |
| **正例**       | 融资轮次；大厂战略重组；API 定价战；市场份额；收购与并购                                   |
| **反例**       | 技术深度评测 → `AI 与模型`；产品 changelog → `产品与工程`；出口管制法条精读 → `社会与政策` |
| **冲突时优先** | vs 社会与政策：公司市场行为 → 商业；立法与公权力 → 社会与政策。vs AI：见 AI 条             |

**轴 C：** 可打 C3 `商业与产业`；人物访谈偏人 → C `人物与访谈`，B 仍可能是商业或人文。

---

### 4. 设计与体验

**定义：** 主线是 **人如何感知与使用界面/品牌/服务**——交互、视觉、设计系统、可用性、体验研究；不是模型能力，也不是纯工程交付。

|                | 例子                                                                                                                             |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **正例**       | 设计系统与组件规范；可用性测试；空状态/信任/认知负荷；品牌与视觉语言；AI 产品的体验原则专文                                      |
| **反例**       | 文生图模型 SOTA → `AI 与模型`（C 可 `图像生成`）；纯前端性能优化 → `产品与工程`；「好看的 AI 壁纸推荐」偏消费内容 → `人文与生活` |
| **冲突时优先** | vs 产品与工程：体验原则/研究 → 设计；功能列表与实现 → 产品。vs AI：体验专文挂 AI 案例仍选 **设计**（C 可空或极少）               |

**轴 C：** 多数 **C 空**；若深度绑定某技术（如语音对话体验）可弱标对应 C，但仍以 B=设计为主航道。

---

### 5. 人文与生活

**定义：** 主线是 **文化、思想、日常与生活实践**——阅读、写作生活、艺术人文、社会文化评论、生活方式；科技若出现也是背景而非硬核主线。

|                | 例子                                                                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **正例**       | 书评与思想随笔；数字生活与习惯；创作者生涯；非技术向的文化现象；「人与工具」的人文讨论                                                    |
| **反例**       | 模型技术拆解 → `AI 与模型`；职场效率工具评测偏生产力软件 → `产品与工程`；设计师谈组件库 → `设计与体验`；政策评论主线是立法 → `社会与政策` |
| **冲突时优先** | vs 设计：生活方式/文化 → 人文；专业设计学科与产物 → 设计。vs AI：「AI 哲学/文明」长文若几乎无技术细节 → 人文；半技术半哲思看主线          |

**轴 C：** 默认 **空**。例外：明确以某 AI 议题为对象的人文评论，可加 1 个 C，但仍认 B=人文与生活。

---

### 6. 社会与政策

**定义：** 主线是 **公共规则与社会治理**——法律、监管、合规框架、公共伦理争议、治理机制（公权力与公共讨论向）。

|                | 例子                                                                                                                     |
| -------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **正例**       | AI 立法与监管动态；出口管制；平台责任与公共伦理辩论；全球治理框架                                                        |
| **反例**       | 模型红队与越狱技术 → B 更可能 `AI 与模型`，C=`安全与对齐`；公司安全合规内控手册无公共规则 → `产品与工程` 或 `商业与产业` |
| **冲突时优先** | vs 商业：监管冲击股价的新闻若主线是市场 → 商业；主线是法案内容 → 社会与政策。vs AI：政策解读无技术 → 社会与政策          |

**轴 C：** 科技监管专题可加 C3 `监管与政策`；纯社会政策非科技可 C 空。

---

### 7. 其他

**定义：** **显式兜底**，不是「杂货抽屉随便扔」。仅当主线无法落入 1–6，或置信不足，或无主线合集。

|                | 例子                                                                    |
| -------------- | ----------------------------------------------------------------------- |
| **正例**       | 多主题日报无 equipoise 主线；领域完全在词表外且不愿误标；模型自身低置信 |
| **反例**       | 懒得判断的 AI 文 → 仍应标 `AI 与模型`；明显融资文 → `商业与产业`        |
| **冲突时优先** | 永远最后选择；有 0.55+ 的明确主线时禁止用其他                           |

**轴 C：** 常空；合集策略与 C 文档一致。

## 4. 近邻易混速查

| 易混对                  | 一句话分开                      |
| ----------------------- | ------------------------------- |
| AI 与模型 ↔ 产品与工程  | 模型/智能技术本身 vs 建造与交付 |
| AI 与模型 ↔ 商业与产业  | 能力与原理 vs 钱与格局          |
| 产品与工程 ↔ 设计与体验 | 功能与实现 vs 感知与体验        |
| 产品与工程 ↔ 商业与产业 | 怎么做出来 vs 怎么赚钱/竞争     |
| 设计与体验 ↔ 人文与生活 | 专业设计/体验学科 vs 文化与日常 |
| 商业与产业 ↔ 社会与政策 | 公司与市场 vs 公权力与公共规则  |
| 人文与生活 ↔ 社会与政策 | 文化生活 vs 治理与立法          |
| 任意 ↔ 其他             | 有主线就不要其他                |

## 5. 组合示例（B + C + A）

| 文章直觉                   | B                        | C（≤3）                  | A（参考） |
| -------------------------- | ------------------------ | ------------------------ | --------- |
| Claude 新模型技术报告      | AI 与模型                | 大模型基础, …            | 发布/论文 |
| Cursor 发布 Agent 编程功能 | 产品与工程               | 编码与开发, Agent 智能体 | 发布      |
| 某模型公司完成融资         | 商业与产业               | 商业与产业               | 快讯      |
| AI 产品空状态与信任设计    | 设计与体验               | （空）或弱相关           | 分析      |
| 播客：创始人谈创业与人生   | 人文与生活 或 商业与产业 | 人物与访谈               | 观点      |
| EU AI Act 条文解读         | 社会与政策               | 监管与政策               | 分析      |
| 本地生活/读书周记          | 人文与生活               | （空）                   | 观点/其他 |
| 30 条 AI 杂烩日报          | 其他 或 AI 与模型\*      | （空）                   | 合集      |

\*合集若明确「只聚合 AI 资讯」且产品希望进 AI 时间线，允许 B=`AI 与模型`；若多领域杂烩 → `其他`。

## 6. 明确不单列（v1 克制）

| 不单列            | 归入                                                                   |
| ----------------- | ---------------------------------------------------------------------- |
| 编程 / 开发       | 产品与工程（C 再细到编码与开发）                                       |
| 科研 / 论文       | 内容属 AI → AI 与模型；否则其他或人文（看对象）；**形态**用 A=论文     |
| 教育 / 医疗等垂直 | 主线是落地 → 产品与工程；主线是政策 → 社会与政策；C 可应用与落地       |
| 游戏 / 娱乐       | 偏产品技术 → 产品与工程；偏文化消费 → 人文与生活                       |
| 安全（信息安全）  | 模型对齐安全 → AI 与模型；企业安全工程 → 产品与工程；立法 → 社会与政策 |

新增 B 词门槛：连续多周稳定供给 + 无法用现有 6 正类表达 + 筛选器里有人会单独勾选。

## 7. 无法分类时怎么走

```
能否判断主线大盘？
  ├─ 能 → 1–6 选一
  └─ 不能 → 其他

是否还要打 C？
  ├─ 主线在 AI 议题树内 → 按 C 词表 0–3 个
  └─ 否则 → C 全部留空（成功，不是失败）
```

**「无法分类」只表示无法细到 C，不表示条目失败。**  
B=`其他` + C 空 + A=`其他` 是合法三元组。
