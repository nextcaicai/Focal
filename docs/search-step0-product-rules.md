## 1. 目标

在不重做检索引擎的前提下，先把搜索从「浮层跳转器」收成：

> **向全部订阅库提问 → 中栏扫结果 → 右侧读正文（列表不丢）。**

---

## 2. 定案清单

### 2.1 入口位置

| 角色                         | 定案                                                                  |
| ---------------------------- | --------------------------------------------------------------------- |
| **主视觉入口**               | **侧栏「查找」分区首行**（与今天/未读/收藏同组，UI 方案 D）           |
| **快捷键 Cmd/Ctrl+K**        | **聚焦侧栏搜索框**（同一套库搜索逻辑）                                |
| **内容页右上角搜索**         | **移除，或改为触发同一 focus**——不再作为库搜索主入口                  |
| **现有 CmdK 浮层（库检索）** | **废弃作为文章/库搜索 UI**；若保留 Command 面板，不承担订阅库全文检索 |

**心智：** 入口在「库 / 导航」层，不在「当前文章工具」层，避免被理解成页内查找。

**文案：**

- Placeholder：`搜索全部订阅…` / `Search all subscriptions…`（i18n 实现时定 key）
- 避免：`搜索…`、暗示「当前源/当前页」的措辞

---

### 2.2 搜索范围

| 项             | 定案                                                                             |
| -------------- | -------------------------------------------------------------------------------- |
| **默认**       | **全库**（所有已订阅源的本地条目）                                               |
| **与侧栏选中** | 默认**不**绑定当前 feed / folder / topic                                         |
| **可选收窄**   | 「仅当前」chip：将范围限制为进入搜索时的 sidebar selection                       |
| **Smart Feed** | Today / Unread 等用**过滤**表达（时间窗 / 未读状态），不单独定义另一套「默认库」 |

**「仅当前」解析：**

| 进入搜索时的 scope | 「仅当前」含义                            |
| ------------------ | ----------------------------------------- |
| 单个源             | 该 `feedId`                               |
| 文件夹             | 文件夹下全部 feed                         |
| Topic              | 该 topic selector 命中集合 ∩ 关键词       |
| All                | 等同全库                                  |
| Today / Unread     | 在全库（或当前集合）上叠加时间 / 未读过滤 |

默认 UI 选中：**全库**。

---

### 2.3 结果形态

| 项                    | 定案                                          |
| --------------------- | --------------------------------------------- |
| **主列表**            | **中栏**展示搜索结果（文章为主）              |
| **点开文章**          | **仅右侧**展示正文；**中栏结果列表保持**      |
| **订阅源命中**        | 次要：折叠区或列表底部，不与文章抢主列表权重  |
| **结果 = 临时 scope** | 中栏处于 `search` scope，标题明示范围与 query |

**搜索态中栏头示例：**

```text
全库搜索「Claude」· 24 篇
[ 全库 | 仅当前 ]   排序：相关 ▾   [ 清除 ]
```

（「关注为 Topic」属后续 Step，本 Step 0 不强制，可预留位置。）

---

### 2.4 进入 / 退出搜索

**进入：**

- 侧栏顶输入并产生有效 query（或提交，实现可选防抖）
- Cmd/Ctrl+K focus 侧栏搜索框

**退出（中栏离开 search scope）：**

| 动作                                             | 结果                                                                        |
| ------------------------------------------------ | --------------------------------------------------------------------------- |
| 清除搜索（✕ / 清空 query）                       | 中栏回到**进入搜索前的 scope**（需记 `previousScope`）                      |
| 点击侧栏其它导航（某源 / Smart Feed / Topic 等） | **退出搜索**，进入该导航 scope                                              |
| Esc                                              | 实现可选：清空搜索并 blur，或仅 blur；**不得**在仍有 query 时静默丢结果会话 |

**禁止：** 仅因选中一篇结果文章而销毁搜索列表 / 关闭检索会话。

---

### 2.5 排序

**默认排序键（稳定、可测）：**

```text
1. 关键词匹配强度（主）
   标题强命中 > 标题包含 > 描述/摘要 > 正文（有则）
2. 同档内：publishedAt 新 → 旧
3. 再同档：qualityScore 高 → 低
   - 无质量分：中性（不沉底、不置顶）
```

**用户可切换：**

| 选项             | 行为                                 |
| ---------------- | ------------------------------------ |
| **相关**（默认） | 上式综合序                           |
| **最新**         | 以时间为主（可仍要求通过关键词过滤） |

「优质」单独排序：非 Step 0/1 必做。

## 5. 验收（Step 1 对照本规则）

- [ ] 侧栏顶有主搜索；placeholder 表达「全部订阅」
- [ ] Cmd+K 聚焦该输入，不打开旧文章检索浮层（或浮层不再搜库）
- [ ] 内容区右上角不再是独立库搜索主入口
- [ ] 有 query 时中栏为结果列表；连续打开多篇文章，中栏结果仍在
- [ ] 默认结果含跨源文章；标题/行上能看出「全库」与来源
- [ ] 清除搜索或点侧栏导航可干净退出
- [ ] 默认排序体感符合：匹配优先，其次新，再次有分时质量

---

## 6. 最短复述

1. **入口在侧栏顶；Cmd+K 聚焦它；内容右上不再主搜。**
2. **默认搜全库；可选仅当前。**
3. **结果在中栏；点文章只换右侧。**
4. **排序：匹配 → 时间 → 质量分。**
5. **重引擎与 RAG 后置。**
