# 实体轴（D）— 延期决策

> 状态：**已决策，目前不做**  
> 日期：2026-07-11  
> 配套真源：A `topic-taxonomy-genre-v1.md` · B `topic-taxonomy-domain-v1.md` · C `topic-taxonomy-topics-v1.md`

---

## 决策

**不将「实体」做成与 A/B/C 同级的、每条 entry 必打的闭集标签轴。**

- 不新增入口级 Entity 字段与全量闭集打标流水线。
- 厂商/产品向阅读若出现，用更轻手段（见下），不先上满血 D。
- 将来若重开，须单独评审，不得 silent 加轴。

---

## 原因（摘要）

| 点             | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 需求真、手段重 | 「按 OpenAI / Claude 追」有价值，但不等于每文 0–3 个闭集实体 |
| 词表难养       | 公司/产品线/别名高频变化，闭集治理成本远高于 A/B/C           |
| 与订阅源重叠   | 官方 Blog 等已提供「按来源」；强制实体 chip 信息增益常偏低   |
| 展示位紧张     | 已定 chip 优先级 C > A > D > B；再上硬 D 易噪                |
| A/B/C 已够骨架 | 缺 D 不阻碍形态/领域/议题体系成立                            |

---

## 现阶段若需要「厂商向」入口

优先顺序（仍属产品可选，**非本决策的实现任务**）：

1. **源绑定** — 某 feed 归入「OpenAI」等集合
2. **关键词 / 别名选择器** — 作 My Topic 或目录预设查询
3. **精选白名单目录卡** — 只展示少数厂商，**不为每条强制打标**

禁止默认路径：自由生成实体字符串写入正式标签。

---

## 何时可重开（弱 D，非满血）

同时接近以下再议：

1. 按厂商读成为与按议题读同级的核心习惯
2. 大量相关内容散落在综合源，源绑定 + 关键词明显不够
3. 愿意维护小规模实体表（别名、合并、废弃）与建议审核队列

若重开，默认形态应为 **弱 D**：小闭集、允许空、主实体优先（0–1）、展示默认不露、主要用于目录/筛选。
